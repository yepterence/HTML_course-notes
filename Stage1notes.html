More concepts about HTML

Google Chrome developer tools

Google Chrome has an in-built functionality that allows the user to investigate the structure of the page and the styles that were being used. It should be noted that the elements are:

  • Rectangular
  • Not all of the elements are available for eg., "head" element
  • All text is same as the text on the page

HTML pages have a tree-like structure where, tags are sub-classified certain sections of the documents. The browser organizes these tags into a tree with the help of DOM (Document Object Model) Each tag can have an attribute with a value, for eg., "class" (attribute)="title"(value).

Cascade Styling Sheets (CSS)

Allows the user to change how the elements will look in the document using a language with specific set of syntax and rules. It helps define the positioning, color, etc., of the elements.

Arrangement of elements in an HTML

All elements are arranged in the form of boxes stacked on top of one another.
Note from expert:It is best to write/draw out your project on paper and isolate the content into boxes. Start from the big boxes and then get to the small boxes.
"<div>"is a great for organizing your content in your html document and the addition of "class" to your div tags makes it easier to style the document when it does become more complex and it also helps by reducing repetition in your document.

The Box Model

All HTML elements can be considered as boxes. In CSS, the term "box model" is used when talking about design and layout. The CSS box model is essentially a box that wraps around every HTML element. It consists of:

  • Content - Centermost element, text/ content, other elements protect it, like a nucleus so that it does not get mixed up with other content on the page
  • Padding - green colored. clears an area around content and is afected by the bg color of the box and is the box that protects content directly
  • border inherits the color property of the box
  • Margin - Outermost, space between boxes.

Code-Test-Refine

In order to streamline web development, the steps stated below can be followed:

  • Define the structure of the page. Classify everything into its "natural" boxes.
  • Look for repeated styles and semantic elements. Input appropriate tags
  • Size of the boxes
  • Position boxes

Once these have been determined. The finer details can be applied to the HTML.
